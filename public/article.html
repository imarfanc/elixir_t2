Building Static Sites with Phoenix and daisyUI

Phoenix Elixir daisyUI

Phoenix Framework combined with daisyUI creates a powerful toolkit for building beautiful static websites with minimal effort.

Why Phoenix for Static Sites?

While Phoenix is known for its real-time capabilities and LiveView, it's also an excellent choice for static site generation. The framework provides:

  • Fast compilation and build times
  • Powerful templating with HEEx
  • Built-in asset pipeline
  • Easy deployment options

The Power of daisyUI

daisyUI is a component library built on top of Tailwind CSS. It provides:

daisyUI components are semantic, accessible, and themeable out of the box!
  1. Pre-built, beautiful components
  2. Multiple theme options
  3. Semantic class names
  4. Full Tailwind CSS compatibility

Getting Started

To create your own static Phoenix site with daisyUI, follow these steps: 

mix phx.new my_app --no-ecto --no-live
cd my_app
mix deps.get
mix tailwind.install

Deployment

Once you've built your site, you can deploy it to GitHub Pages, Netlify, Vercel, or any static hosting service. The build process generates a public/ directory with all your static assets ready to serve.

Pro Tip

Use GitHub Actions to automate your deployment process. Every push to main can trigger a new build and deployment!

Conclusion

Phoenix and daisyUI make a fantastic combination for static site generation. You get the power and flexibility of Phoenix with the beautiful, pre-built components of daisyUI. Happy building!
